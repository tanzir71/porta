# Porta 1.2.0

Porta 1.2 keeps its account-free Cloudflare Quick Tunnel default and adds a
provider layer you can control from the app. Existing 1.1 shares migrate
automatically and continue using Cloudflare Quick unless you choose otherwise.

## Downloads

- **Windows 10/11 x64:** `Porta_1.2.0_x64-setup.exe`
- **Apple silicon macOS:** `Porta_1.2.0_aarch64.dmg`
- Verify the matching `.sha256` attachment before overriding an operating-system warning.

## What’s new

- Add managed Cloudflare Tunnel, ngrok, or custom command profiles in Settings.
- Choose a global default provider and optionally override it per share.
- Select provider executables with the native file picker and test each profile before using it.
- Keep tunnel tokens in Keychain or Credential Manager, never in Porta’s JSON store or command arguments.
- Run custom CLIs directly without a shell using one argument per line and `{origin}`, `{host}`, or `{port}` placeholders.
- Normalize provider-specific visitor headers for Porta’s local stats.

Managed Cloudflare and ngrok profiles require the relevant vendor account and
configuration. Custom profiles require a compatible tunnel executable and an
HTTPS public endpoint. Those providers’ limits, terms, and privacy practices
apply independently of Porta. Only select executables you trust; Porta launches
the chosen CLI with your user account’s permissions.

## Windows installation

The Windows installer is not Authenticode signed, so Microsoft Defender
SmartScreen may show **Windows protected your PC**.

1. Run `Porta_1.2.0_x64-setup.exe`.
2. Select **More info**.
3. Confirm the file came from the official Porta GitHub Release and that its
   SHA-256 matches the attached checksum.
4. Select **Run anyway**.

The per-user installer does not require administrator access.

## macOS installation

The macOS build is ad-hoc signed and not notarized. Open the DMG, drag Porta to
Applications, and try opening it once. If macOS blocks it, open **System
Settings → Privacy & Security**, choose **Open Anyway**, and confirm **Open**.

## Security reminder

A live Porta URL is public to anyone who has it. Use password protection for
sensitive folders, disable uploads unless needed, and stop the share when you
are done. Porta has no account, analytics, telemetry, or hosted file service.
